Class PMap<K, V>

PMap implements builtin Map interface. PMap transforms a key into a string and stores the original key-value pair internally with the transformed key into a native Map.

Example usage:

class Position {
  constructor(public x: int, public y: int) {}
  static toString(p: Posi) { return `${p.x}-${p.y}`; }
}

const map = new PMap<Position, string>(P.toString);
const myKey = new Position(1, 2);
map.set(myKey, 'here is a treasure hidden');
console.log(map.get(myKey)); // 'here is a treasure hidden'
console.log(map.get(new Position(1, 2))); // 'here is a treasure hidden'

As PMap uses internally a native Map, sublinear average access is guaranteed (ECMAScript 2015) - but it should be a safe guess, that all modern JS engines provide highly optimized Maps with O(1) average access.
